## 二、三级记忆系统流程

### 记忆存储结构

```
Redis 键结构
│
├── memory:global                         ← 全局运维经验 (≤1500字)
├── memory:admin:{admin_id}               ← 管理员档案 (≤1500字)
├── memory:server:{server_id}             ← 服务器专属信息 (≤1000字)
│
├── memory:backup:global                  ← 全局记忆备份 (LIFO, 最多10版)
├── memory:backup:admin:{admin_id}        ← 管理员记忆备份
├── memory:backup:server:{server_id}      ← 服务器记忆备份
│
├── session:active:{admin}:{server}       ← 会话活跃状态
└── session:active_set                    ← 活跃会话集合 (用于扫描)
```

### 记忆手动 CRUD 流程

```
┌─────────────────────────────────────────────────────┐
│  记忆 API 端点 (/api/v1/memory)                      │
│                                                     │
│  读取:                                               │
│  GET /memory/global              → 获取全局记忆       │
│  GET /memory/admin/{admin_id}    → 获取管理员记忆     │
│  GET /memory/server/{server_id}  → 获取服务器记忆     │
│                                                     │
│  写入:                                               │
│  PUT /memory/global              → 更新全局 (仅管理员) │
│  PUT /memory/admin/{admin_id}    → 更新管理员记忆     │
│  PUT /memory/server/{server_id}  → 更新服务器记忆     │
│                                                     │
│  备份:                                               │
│  GET  /memory/backup/{type}/{id} → 查看备份列表       │
│  POST /memory/rollback/{type}/{id} → 回滚到指定版本   │
└─────────────────────────────────────────────────────┘
│
│  写入流程:
▼
┌──────────────────────────────┐
│  ① 字数限制校验               │
│  全局/管理员 ≤ 1500字         │
│  服务器 ≤ 1000字              │
│  超限 → 返回错误              │
└──────────────────────────────┘
│
▼
┌──────────────────────────────┐
│  ② 自动备份旧版本             │
│  Redis LPUSH 到 backup 列表  │
│  LTRIM 保留最近 10 个版本     │
└──────────────────────────────┘
│
▼
┌──────────────────────────────┐
│  ③ 写入新内容到 Redis         │
│  JSON: {content, updated_at} │
└──────────────────────────────┘
```
