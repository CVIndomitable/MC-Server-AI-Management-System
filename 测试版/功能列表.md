## 📱 客户端

### iOS 原生客户端 (mc-admin-ios)
- **登录与认证**
  - JWT Token 认证
  - 多账号保存与快速切换
  - 自动登录（Token 未过期时）
  
- **服务器管理**
  - 服务器列表展示
  - 服务器绑定（输入 server_id）
  - 服务器解绑
  - 服务器重命名
  - 服务器权限审批（owner 审批其他管理员加入）
  
- **AI 对话 (ChatView)**
  - 自然语言指令输入
  - Markdown 格式渲染（代码块、列表、粗体等）
  - 多轮对话历史（每服务器保留 50 条）
  - 三级模型路由选择（flash / standard / pro）
  - 仅查询模式开关（AI 只给建议，不执行操作）
  - 消息发送状态指示
  - 错误提示与重试
  
- **实时状态监控 (StatusView)**
  - TPS 实时显示（含 Spark 多窗口 TPS）
  - MSPT（每 tick 毫秒数）及百分位统计
  - CPU 使用率（进程 + 系统）
  - 内存使用情况（已用 / 总量 / 百分比）
  - 在线玩家列表
  - 最近错误日志（WARN/ERROR 级别）
  - GC 统计信息（收集器、频率、耗时）
  - 自动刷新（5 秒间隔）
  
- **快捷操作 (ActionsView)**
  - 一键重启服务器
  - 踢出玩家
  - 授予 OP 权限
  - 移除 OP 权限
  - 发送全服公告
  - 执行自定义指令
  
- **档案馆 (ArchiveView)**
  - Spark Profiler 采样历史列表
  - 采样详情查看（开始/停止时间、命令、状态）
  - AI 深度分析报告（含扩展思考过程）
  - 性能剖析原始数据查看
  - 采样状态筛选（进行中 / 已完成 / 失败）
  - 分页加载
  
- **LLM 供应商管理 (ProvidersView)**
  - 供应商列表（名称、base_url、优先级、状态）
  - 添加新供应商
  - 编辑供应商配置
  - 删除供应商
  - 启用/禁用供应商
  - 优先级调整（数字越大优先级越高）
  - 自动故障转移（主供应商失败时切换到备用）
  
- **设置 (SettingsView)**
  - 账号信息展示
  - 修改密码
  - 退出登录
  - 用户管理（仅管理员）
    - 注册新用户
    - 删除用户
    - 重置用户密码
  - 服务器管理（重命名、解绑）

### Expo 跨平台客户端 (mc-admin-client)
- 支持 iOS / Android / Web 三端
- 功能与 iOS 原生客户端基本一致
- 基于 React Native + Expo SDK 54

## 🖥️ 后端服务 (mc-admin-server)

### 用户与权限系统
- **用户认证**
  - 用户注册（仅管理员可创建）
  - 用户登录（JWT Token，默认 7 天有效期）
  - 修改密码
  - 重置密码（管理员操作）
  - 用户删除（管理员操作）
  
- **权限管理**
  - 系统角色：`admin`（管理员）/ `user`（普通用户）
  - 服务器角色：`owner`（首个绑定者）/ `admin`（审批加入）
  - 用户-服务器绑定系统
  - 服务器访问权限隔离
  - 审批流程（owner 审批其他管理员加入）

### AI 智能体
- **自然语言理解**
  - 用户意图识别
  - 自动转换为 MC 指令
  - 多轮对话上下文保持（每服务器 50 条历史）
  
- **三级模型路由**
  - **flash**：简单操作（踢人、授权、查状态）
  - **standard**：复杂问题（解释、建议、多步骤操作）
  - **pro**：深度分析（性能诊断、崩溃排查、优化方案）
  - 自动路由（基于关键词和消息长度）
  - 手动指定模型等级
  
- **仅查询模式**
  - AI 只给建议和指令，不执行任何操作
  - 安全观察服务器状态
  - 学习 MC 指令用法
  
- **Function Calling 工具**
  - `execute_command` — 执行任意 MC 指令
  - `kick_player` — 踢出玩家
  - `op_player` — 授予 OP 权限
  - `deop_player` — 移除 OP 权限
  - `get_status` — 获取服务器状态
  - `read_log` — 读取服务器日志（支持行数和关键字过滤）
  - `restart_server` — 重启服务器
  - `broadcast` — 发送全服公告

### Spark 性能分析集成
- **自动数据上报**
  - TPS（10s / 1m / 5m / 15m 多窗口）
  - MSPT（均值、P50、P95、最大值）
  - CPU 使用率（进程 + 系统）
  - GC 统计（收集器、频率、耗时）
  
- **AI 指令支持**
  - `/spark tps` — 详细 TPS 和 CPU
  - `/spark ping` — 玩家延迟
  - `/spark health` — 完整健康报告
  - `/spark profiler start/stop` — CPU 性能剖析
  - `/spark tickmonitor` — Tick 耗时监控
  - `/spark gc` — GC 历史记录
  - `/spark heapsummary` — 堆内存摘要
  
- **卡顿诊断流程**
  - AI 自动识别卡顿问题
  - 按流程执行：状态检查 → tick 监控 → 针对性 profiler → 分析报告
  - 常见原因识别（WorldEdit、区块生成、实体过多、红石机器等）

### 档案馆系统
- **Spark Profiler 采样归档**
  - 自动记录每次 profiler start/stop
  - 保存采样时间、命令参数、输出结果
  - 存储性能剖析原始数据和报告链接
  
- **AI 深度分析**
  - 自动触发 AI 分析（使用 pro 模型 + extended thinking）
  - 分析内容：性能瓶颈、优化建议、根因诊断
  - 保存扩展思考过程（thinking）和最终分析报告
  - 记录使用的模型和供应商
  
- **历史查询**
  - 按服务器筛选
  - 按状态筛选（进行中 / 已完成 / 失败）
  - 分页加载
  - 详情查看（含 AI 分析和原始数据）

### 三级记忆系统
- **全局记忆**
  - 跨服务器通用知识
  - 系统级运维经验
  
- **管理员记忆**
  - 个人运维偏好
  - 常用操作习惯
  
- **服务器记忆**
  - 服务器特定配置
  - 历史问题记录
  
- **记忆管理**
  - CRUD 操作（创建、读取、更新、删除）
  - 自动注入到 AI System Prompt
  - 后台自动整理（基于对话历史提取关键信息）

### 命令缓存系统
- **Redis 缓存**
  - 相同消息跳过大模型调用
  - LFU 淘汰策略
  - 默认 1 小时 TTL
  - 降低延迟和 API 成本
  
- **缓存键生成**
  - 基于用户输入 + 服务器状态 + 记忆内容
  - 自动去重

### 命令安全审核
- **风险评估**
  - 三级风险：`low` / `medium` / `high`
  - 基于指令类型和上下文自动评估
  - 危险操作（stop、kill、删除等）标记为高风险
  
- **AI 二次审核**
  - 高风险指令触发 AI 审核
  - 分析指令意图和潜在影响
  - 给出审核建议和风险提示
  
- **审核结果**
  - `approved` — 通过
  - `rejected` — 拒绝
  - `needs_confirmation` — 需要用户确认

### LLM 供应商池
- **多供应商管理**
  - 支持多个 LLM 供应商（Anthropic API 兼容）
  - 自定义 base_url 和 API key
  - 优先级排序（数字越大优先级越高）
  - 启用/禁用开关
  
- **自动故障转移**
  - 主供应商失败时自动切换到备用
  - 按优先级依次尝试
  - 记录失败原因和使用的供应商
  
- **供应商 CRUD**
  - 创建、读取、更新、删除供应商
  - 仅管理员可操作

### 服务器管理
- **服务器注册**
  - 模组首次连接自动注册
  - 生成唯一 server_id
  
- **服务器绑定**
  - 用户通过 server_id 绑定服务器
  - 首个绑定者自动成为 owner
  - 其他用户需 owner 审批
  
- **服务器信息**
  - 服务器列表
  - 在线状态
  - 管理员列表
  - 重命名
  - 解绑

### WebSocket 管理
- **模组连接**
  - 模组主动出站连接
  - Token 认证
  - 心跳保活
  - 自动重连
  
- **消息路由**
  - 状态上报（TPS、内存、CPU、玩家、日志、Spark 数据）
  - 指令下发（execute、kick、op、deop、restart、broadcast、read_log）
  - 执行结果回传
  
- **连接管理**
  - 僵死连接自动清理（5 分钟无心跳）
  - 连接状态监控

### 安全机制
- **认证与鉴权**
  - JWT Token 认证
  - bcrypt 密码加密
  - Token 过期自动刷新
  
- **输入验证**
  - 请求体大小限制（1MB）
  - SQL 注入防护
  - XSS 防护
  
- **Prompt 注入防护**
  - 用户输入清洗（移除 `<system>`、`<user>` 等标签）
  - 服务器状态数据清洗（移除潜在注入内容）
  - 记忆内容隔离
  
- **CORS 配置**
  - 允许的源配置
  - 禁用 credentials（Bearer Token 认证不走 cookie）

### 健康检查
- **健康端点** (`/health`)
  - API 状态
  - Redis 连接状态
  - 数据库连接状态
  - 整体健康状态（ok / degraded）

## 🎮 Minecraft 模组 (mc-admin-mod)

### 支持版本
- **Fabric 1.20** (mc-admin-mod-fabric-1.20)
- **NeoForge 1.21.1** (mc-admin-mod-forge-1.21.1)

### 核心功能
- **WebSocket 连接**
  - 使用 Java 内置 `java.net.http.WebSocket`（无外部依赖）
  - 主动出站连接云服务器
  - Token 认证
  - 自动重连（指数退避，最大 60 秒）
  - 心跳保活（30 秒间隔）
  
- **状态上报**
  - 每 5 秒自动上报
  - TPS（基础 + Spark 多窗口）
  - 内存使用（已用 / 最大 / 百分比）
  - CPU 使用率（进程 + 系统）
  - 在线玩家列表
  - 最近错误日志（WARN/ERROR 级别，最多 10 条）
  - Spark 性能数据（TPS、MSPT、CPU、GC）
  
- **指令执行**
  - 接收云服务器下发的指令
  - 主线程执行（通过 `ServerTickEvent` 调度）
  - 命令白名单（可配置）
  - 执行结果回传（成功/失败 + 输出）
  
- **日志收集**
  - 拦截 WARN/ERROR 级别日志
  - 自动上报到云服务器
  - 最多保留 10 条最近日志
  
- **日志读取**
  - 读取 `logs/latest.log` 文件
  - 支持按行数读取（最近 N 行）
  - 支持关键字过滤
  - 用于排查报错、追溯玩家行为、命令执行记录等
  
- **Spark 集成**
  - 自动检测 Spark 模组是否安装
  - 收集 Spark 提供的详细性能数据
  - 安全封装（Spark 不存在时不影响模组运行）
  - 支持所有 Spark 命令（通过 `execute_command` 调用）

### 游戏内 AI 对话
- **`/ai` 命令**
  - 玩家在游戏内直接与 AI 对话
  - 自动创建会话（首次使用）
  - 多轮对话支持
  - 回复显示在聊天框
  
- **聊天拦截**
  - 配置前缀（如 `@ai`）触发 AI 对话
  - 无需输入命令，自然对话
  
- **会话管理**
  - 每玩家独立会话
  - 会话超时自动清理（30 分钟无活动）
  - 会话历史保留（最多 20 条）

### 客户端功能（可选）
- **TPS HUD 显示**
  - 屏幕左上角显示实时 TPS
  - 颜色指示（绿色 > 18 / 黄色 > 15 / 红色 ≤ 15）
  - 快捷键切换显示（默认 `K`）
  
- **快捷键**
  - 可自定义按键绑定
  - 支持组合键

### 配置管理
- **配置文件** (`config/mcadmin-mod.json`)
  - WebSocket 服务器地址
  - 认证 Token
  - Server ID
  - 命令白名单
  - 日志级别
  - 心跳间隔
  - 重连策略
  
- **热重载**
  - 配置修改后自动重载（部分配置需重启）

## 📝 已知问题与限制

1. **Redis 版本兼容性**
   - 中途岛 Redis 5.0.3 不支持 `GETDEL` 等 6.2+ 命令
   - 已改用 `GET` + `DEL` 替代

2. **模组外部依赖**
   - NeoForge 模组无法使用外部 WebSocket 库
   - 已改用 JDK 内置 `java.net.http.WebSocket`

3. **Spark 初始化时机**
   - Spark API 需在服务器启动后才可用
   - 已实现延迟检测和重试机制

4. **客户端硬编码地址**
   - 单部署项目，后端地址硬编码为 `47.113.221.26`
   - 刻意设计，避免配置复杂度

5. **档案馆 AI 分析成本**
   - 使用 pro 模型 + extended thinking
   - 单次分析约 10-20k tokens
   - 建议按需触发，避免频繁分析

## 🔐 安全注意事项
